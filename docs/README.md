## 📝 Documentation Standards

### When to Update Documentation

- **New Feature**: Create/update feature documentation
- **API Change**: Update Swagger annotations and README
- **Breaking Change**: Update all relevant docs and CHANGELOG
- **Bug Fix**: Update if it affects documented behavior

### Documentation Guidelines

1. **Be Concise**: Keep docs brief but complete
2. **Use Examples**: Include code examples
3. **Keep Updated**: Update docs with code changes
4. **Link Related**: Cross-reference related documentation
5. **Format Consistently**: Follow existing structure
